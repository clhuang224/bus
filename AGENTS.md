# Project Conventions

- Use English for developer-facing console output such as `console.warn`, `console.error`, and logs.
- Use Traditional Chinese for user-facing UI copy such as alerts, empty states, and inline warnings.
- Follow Conventional Commits for commit messages.
- Format commit subjects as `<type>[optional scope]: <description>`.
- Use `feat` for new user-facing functionality.
- Use `fix` for bug fixes and behavior corrections.
- Other common types are allowed when appropriate, such as `docs`, `refactor`, `test`, `chore`, `build`, `ci`, `style`, and `perf`.
- Use an optional scope when it adds useful context, such as `fix(nearby): ...` or `refactor(geolocation): ...`.
- Keep the subject concise and imperative.
- Add a blank line before the body when extra context is needed.
- Mark breaking changes with `!` in the header or a `BREAKING CHANGE:` footer.
